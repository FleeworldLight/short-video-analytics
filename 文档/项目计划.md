# 短视频平台内容分析系统 · 详细实施计划

基于 **KuaiRec 2.0** 数据集  
**技术栈**：Hadoop HDFS + Hive + Spark + MySQL + Spring Boot + ECharts  
**基础设施**：Docker 单节点
---

## 一、数据集说明

KuaiRec（快手真实用户行为数据）

- 收录时间：2020-07 至 2020-09
- 论文：*KuaiRec: A Fully-observed Dataset for Insights for Evaluating Recommender Systems*

### 数据规模

| 数据集 | 用户数 | 视频数 | 交互数 | 密度 |
|--------|--------|--------|--------|------|
| 小矩阵（small_matrix） | 1,411 | 3,327 | 4,676,570 | 99.6% |
| 大矩阵（big_matrix） | 7,176 | 10,728 | 12,530,806 | 16.3% |

### 文件列表

| 文件 | 说明 |
|------|------|
| `big_matrix.csv` | 用户-视频交互主表（8 列） |
| `item_categories.csv` | 视频分类标签（video_id, feat） |
| `item_daily_features.csv` | 视频日级统计（57 列，含作者/播放/点赞/评论/分享等） |
| `user_features.csv` | 用户画像特征（31 列，含活跃度/关注数/粉丝数/18维 one-hot） |


### 字段映射

| 试卷要求字段 | KuaiRec 实际字段 | 处理方式 |
|-------------|-----------------|---------|
| user_id | user_id | 直接使用 |
| video_id | video_id | 直接使用 |
| watch_duration | play_duration（毫秒） | 转为秒 |
| video_duration | video_duration（毫秒） | 转为秒 |
| timestamp | timestamp 或 time | 直接使用 |
| like_flag | 无直接字段 | watch_ratio > 2.0 → 1 |
| comment_flag | 无直接字段 | 可从 item_daily_features 推算 |
| share_flag | 无直接字段 | 同上 |
| uploader_id | item_daily_features.author_id | 关联获取 |
| category | item_categories.feat | 直接使用 |

> **注**：KuaiRec 没有 per-interaction 的点赞/评论/分享标记，watch_ratio > 2.0 视为"喜欢"。

## 三、数仓分层设计

### ODS 层（原始数据，外部表，CSV SerDe 直接加载）

| 表名 | 来源 | 列数 |
|------|------|------|
| `ods_raw_interaction` | big_matrix.csv | 8 列 |
| `ods_raw_item_category` | item_categories.csv | 2 列 |
| `ods_raw_item_daily_features` | item_daily_features.csv | 57 列 |
| `ods_raw_user_feature` | user_features.csv | 31 列 |

### DWD 层（清洗明细，ORC + Snappy，按日期分区）

`dwd_interaction_detail`

清洗逻辑：

1. 过滤 play_duration <= 0 或 video_duration <= 0
2. 过滤 play_duration > video_duration \* 3（异常值）
3. 去重（user_id + video_id + date）
4. play_duration / 1000 → play_duration_sec
5. watch_ratio >= 1.0 → completion_flag = 1
6. watch_ratio > 2.0 → like_flag = 1
7. 时间戳 → event_date / event_hour / weekday / time_period
8. 关联视频分类标签

### DWS 层（轻度汇总，ORC + Snappy，按日期分区）

| 表名 | 粒度 |
|------|------|
| `dws_user_daily_agg` | 用户ID、日期、观看视频数、总观看时长、完播数、平均完播率、点赞数 |
| `dws_video_daily_agg` | 视频ID、日期、播放量、总观看时长、完播量、平均完播率、热度分 |
| `dws_category_daily_agg` | 分类标签、日期、播放量、完播数、平均完播率 |
| `dws_user_weekly_agg` | 用户ID、周起始、活跃天数、总播放数、总观看时长、完播数、平均完播率 |

### ADS 层（指标结果表，HDFS TEXTFILE + MySQL）

| 表名 | 说明 |
|------|------|
| `ads_completion_rate_by_category` | 按类别完播率 |
| `ads_completion_rate_by_author` | 按创作者完播率 |
| `ads_user_retention` | 留存率（次日/7日/30日） |
| `ads_content_hot_ranking` | 内容热度排行 Top 50 |
| `ads_influencer_index` | 创作者影响力指数 Top 20 |
| `ads_time_period_analysis` | 时段播放分析 |

### 维度表

| 表名 | 说明 | 来源 |
|------|------|------|
| `dim_user` | 用户维度（user_active_degree, follow_user_num, register_days 等） | user_features.csv |
| `dim_video` | 视频维度（video_id, duration_sec, category_list, uploader_id） | big_matrix + item_categories |
| `dim_category` | 分类维度（tag_id, tag_name, category_group） | 手动定义（16 个标签分组） |
| `dim_date` | 时间维度（程序生成） | 程序生成 |
| `dim_creator` | 创作者维度（creator_id, total_videos, distinct_categories, avg_video_duration） | item_daily_features |

## 五、KPI 指标计算方案

### KPI 1：完播率分布

**按类别：**

```sql
INSERT OVERWRITE TABLE ads_completion_rate_by_category
SELECT c.tag_name,
       COUNT(*)              AS total_plays,
       SUM(d.completion_flag) AS total_completions,
       ROUND(SUM(d.completion_flag) / COUNT(*), 4) AS completion_rate
FROM dwd_interaction_detail d
JOIN ods_raw_item_category ic ON d.video_id = ic.video_id
LATERAL VIEW explode(split(ic.feat, ',')) t AS tag_str
JOIN dim_category c ON c.tag_id = CAST(tag_str AS INT)
GROUP BY c.tag_name
ORDER BY completion_rate DESC;
```

**按创作者：**

```sql
INSERT OVERWRITE TABLE ads_completion_rate_by_author
SELECT v.uploader_id,
       COUNT(*)                AS total_plays,
       ROUND(AVG(d.completion_flag), 4) AS avg_completion_rate
FROM dwd_interaction_detail d
JOIN dim_video v ON d.video_id = v.video_id
WHERE v.uploader_id IS NOT NULL
GROUP BY v.uploader_id
ORDER BY avg_completion_rate DESC;
```

### KPI 2：用户留存率（次日/7日/30日）

```sql
WITH active_users AS (
  SELECT DISTINCT user_id, dt AS event_date FROM dwd_interaction_detail
)
INSERT OVERWRITE TABLE ads_user_retention
SELECT a.event_date               AS report_date,
       COUNT(DISTINCT a.user_id)  AS dau,
       ROUND(COUNT(DISTINCT CASE WHEN date_diff(b.event_date, a.event_date)=1  THEN a.user_id END) / COUNT(DISTINCT a.user_id), 4) AS day1_retention,
       ROUND(COUNT(DISTINCT CASE WHEN date_diff(b.event_date, a.event_date)=7  THEN a.user_id END) / COUNT(DISTINCT a.user_id), 4) AS day7_retention,
       ROUND(COUNT(DISTINCT CASE WHEN date_diff(b.event_date, a.event_date)=30 THEN a.user_id END) / COUNT(DISTINCT a.user_id), 4) AS day30_retention
FROM active_users a
LEFT JOIN active_users b ON a.user_id = b.user_id
GROUP BY a.event_date
ORDER BY a.event_date;
```

### KPI 3：内容热度排行

热度分 = 完播率×0.35 + 平均观看比例×0.25 + LOG(播放量+1)×0.20 + 互动率×0.20

```sql
INSERT OVERWRITE TABLE ads_content_hot_ranking
SELECT ROW_NUMBER() OVER (ORDER BY hot_score DESC) AS rank_no,
       video_id, hot_score, dt
FROM (
  SELECT video_id, dt,
    ROUND(AVG(completion_flag)*0.35 + AVG(watch_ratio)*0.25
        + LOG(COUNT(*)+1)*0.20 + AVG(like_flag)*0.20, 4) AS hot_score
  FROM dwd_interaction_detail
  GROUP BY video_id, dt
) t
ORDER BY hot_score DESC
LIMIT 50;
```

### KPI 4：创作者影响力指数

影响力分 = avg_completion×0.3 + avg_like×0.3 + LOG(播放量+1)×0.4

```sql
INSERT OVERWRITE TABLE ads_influencer_index
SELECT ROW_NUMBER() OVER (ORDER BY influence_score DESC) AS rank_no,
       uploader_id, total_plays, avg_completion, avg_interaction, influence_score
FROM (
  SELECT v.uploader_id,
         COUNT(*)                    AS total_plays,
         ROUND(AVG(d.completion_flag), 4) AS avg_completion,
         ROUND(AVG(d.like_flag), 4)       AS avg_interaction,
         ROUND(AVG(d.completion_flag)*0.3 + AVG(d.like_flag)*0.3
             + LOG(COUNT(*)+1)*0.4, 4)    AS influence_score
  FROM dwd_interaction_detail d
  JOIN dim_video v ON d.video_id = v.video_id
  WHERE v.uploader_id IS NOT NULL
  GROUP BY v.uploader_id
) t
ORDER BY influence_score DESC
LIMIT 20;
```

### 附加：时段播放分析

```sql
INSERT OVERWRITE TABLE ads_time_period_analysis
SELECT time_period,
       COUNT(*)              AS play_count,
       ROUND(AVG(watch_ratio), 4) AS avg_watch_ratio,
       SUM(like_flag)        AS like_count
FROM dwd_interaction_detail
GROUP BY time_period
ORDER BY play_count DESC;
```
