## 方式一：GitHub Actions CI（推荐）

代码推送到 GitHub 后自动触发 CI 流水线，完成数据下载 → ETL → 可视化 → 部署全流程。

### 触发方式

| 工作流 | 触发事件 | 矩阵数据 | Pages 部署 |
|--------|---------|---------|-----------|
| `CI - 小矩阵 + Pages 部署` | `git push main` 或手动 | `small_matrix.csv` | ✅ |
| `CI - 大矩阵 + Pages 部署` | 仅手动 `workflow_dispatch` | `big_matrix.csv` | ✅ |

### 手动触发步骤

1. 打开 GitHub 仓库 → **Actions** 选项卡
2. 选择 **CI - 大矩阵 + Pages 部署** 或 **CI - 小矩阵 + Pages 部署**
3. 点击 **Run workflow** → 可选勾选 "跳过 ETL+KPI 全流程"（仅部署 Pages，使用模拟数据）
4. 等待执行完成（大矩阵约 30-60 分钟，小矩阵更快）

### 结果产物

- **GitHub Pages**：自动部署到 `https://<用户名>.github.io/<仓库名>/`
- **Artifacts**：可在 Action 运行结果中下载 `results`（CSV + JSON 数据）

## 方式二：本地 Docker + Hadoop 集群运行

在本地或服务器上通过 Docker Compose 启动 Hadoop 集群，手动执行 ETL 流水线。

### 前置要求

- Docker & Docker Compose
- 至少 8GB 可用内存（推荐 16GB）
- KuaiRec 数据集（CI 会自动下载，也可手动下载）
- Java 8+、Python 3

### 步骤

#### 1. 启动集群

```bash
docker compose -f docker/docker-compose.yml up -d \
  namenode datanode resourcemanager nodemanager \
  hive-db hive-metastore hive-server mysql
```

#### 2. 等待 HDFS 就绪并创建目录

```bash
docker exec namenode hdfs dfs -mkdir -p \
  /user/hive/warehouse \
  /data/short_video/raw/ods_interaction \
  /data/short_video/raw/ods_item_category \
  /data/short_video/raw/ods_user_feature \
  /data/short_video/raw/ods_item_daily \
  /data/short_video/raw/ods_social_network \
  /data/short_video/dwd \
  /data/short_video/dws \
  /data/short_video/ads
```

#### 3. 上传数据到 HDFS

```bash
# 从 KuaiRec 数据集目录复制 CSV（假设已在 ./kuairc/data/ 下）
docker cp kuairc/data/big_matrix.csv namenode:/tmp/
docker exec namenode hdfs dfs -put -f /tmp/big_matrix.csv /data/short_video/raw/ods_interaction/

docker cp kuairc/data/item_categories.csv namenode:/tmp/
docker exec namenode hdfs dfs -put -f /tmp/item_categories.csv /data/short_video/raw/ods_item_category/

docker cp kuairc/data/user_features.csv namenode:/tmp/
docker exec namenode hdfs dfs -put -f /tmp/user_features.csv /data/short_video/raw/ods_user_feature/

docker cp kuairc/data/item_daily_features.csv namenode:/tmp/
docker exec namenode hdfs dfs -put -f /tmp/item_daily_features.csv /data/short_video/raw/ods_item_daily/

docker cp kuairc/data/social_network.csv namenode:/tmp/
docker exec namenode hdfs dfs -put -f /tmp/social_network.csv /data/short_video/raw/ods_social_network/
```

#### 4. 执行建表

```bash
docker exec hive-server beeline -u jdbc:hive2://localhost:10000 -n hive -f /scripts/2_create_tables.hql
```

#### 5. 数据清洗（ODS → DWD）

```bash
docker exec hive-server beeline -u jdbc:hive2://localhost:10000 -n hive -f /scripts/1_clean_data.hql
```

#### 6. 构建维度表

```bash
docker exec hive-server beeline -u jdbc:hive2://localhost:10000 -n hive -e "
  INSERT OVERWRITE TABLE dim_category VALUES
    (1, 'Gaming', '娱乐'), (2, 'Music', '艺术'), (5, 'Fashion', '生活'),
    (6, 'Entertainment', '娱乐'), (7, 'Sports', '生活'), (9, 'Comedy', '娱乐'),
    (11, 'Life', '生活'), (12, 'Food', '生活'), (13, 'Agriculture', '生活'),
    (17, 'Animals', '生活'), (18, 'Cars', '生活'), (20, 'Film', '艺术'),
    (26, 'Photography', '文化'), (27, 'Other', '其他'), (28, 'News', '文化'),
    (31, 'Knowledge', '文化');
  INSERT OVERWRITE TABLE dim_creator
  SELECT author_id AS creator_id,
         COUNT(DISTINCT video_id) AS total_videos,
         COUNT(DISTINCT video_tag_id) AS distinct_categories,
         ROUND(AVG(video_duration), 2) AS avg_video_duration
  FROM ods_raw_item_daily_features
  WHERE author_id IS NOT NULL
  GROUP BY author_id;
"
```

#### 7. 执行 KPI 指标计算

```bash
docker exec hive-server beeline -u jdbc:hive2://localhost:10000 -n hive -f /scripts/3_kpi_completion_rate.hql
docker exec hive-server beeline -u jdbc:hive2://localhost:10000 -n hive -f /scripts/4_kpi_retention.hql
docker exec hive-server beeline -u jdbc:hive2://localhost:10000 -n hive -f /scripts/5_kpi_hot_ranking.hql
docker exec hive-server beeline -u jdbc:hive2://localhost:10000 -n hive -f /scripts/6_kpi_influencer.hql
docker exec hive-server beeline -u jdbc:hive2://localhost:10000 -n hive -e "
  INSERT OVERWRITE TABLE ads_time_period_analysis
  SELECT time_period, COUNT(*) AS play_count,
         ROUND(AVG(watch_ratio), 4) AS avg_watch_ratio,
         SUM(like_flag) AS like_count
  FROM dwd_interaction_detail
  GROUP BY time_period
  ORDER BY play_count DESC;
"
```

#### 8. 导出数据为 CSV

```bash
docker exec hive-server beeline -u jdbc:hive2://localhost:10000 -n hive --outputformat=csv2 \
  -e "SELECT * FROM ads_completion_rate_by_category" > 结果数据/completion_rate_by_category.csv
docker exec hive-server beeline -u jdbc:hive2://localhost:10000 -n hive --outputformat=csv2 \
  -e "SELECT * FROM ads_completion_rate_by_author" > 结果数据/completion_rate_by_author.csv
docker exec hive-server beeline -u jdbc:hive2://localhost:10000 -n hive --outputformat=csv2 \
  -e "SELECT * FROM ads_user_retention" > 结果数据/retention.csv
docker exec hive-server beeline -u jdbc:hive2://localhost:10000 -n hive --outputformat=csv2 \
  -e "SELECT * FROM ads_content_hot_ranking" > 结果数据/hot_ranking.csv
docker exec hive-server beeline -u jdbc:hive2://localhost:10000 -n hive --outputformat=csv2 \
  -e "SELECT * FROM ads_influencer_index" > 结果数据/influencer.csv
docker exec hive-server beeline -u jdbc:hive2://localhost:10000 -n hive --outputformat=csv2 \
  -e "SELECT * FROM ads_time_period_analysis" > 结果数据/time_period_analysis.csv
```

#### 9. 转换 CSV 为 JSON（供 Pages 使用）

```bash
python 脚本/csv_to_json.py
```

#### 10. 启动可视化仪表板

参见 [方式三](#方式三仅启动可视化仪表板)。

## 方式三：仅启动可视化仪表板

如果已有分析数据（CSV 或 JSON），可直接启动仪表板。

### 方法 A：Spring Boot 仪表板（连接 MySQL）

```bash
cd 可视化源码/dashboard
mvn spring-boot:run
```

启动后访问 `http://localhost:8080`。

### 方法 B：静态 Pages 仪表板（推荐）

1. 确保 `docs/data.json` 已通过 CI 生成或手动放置
2. 直接双击 `docs/index.html` 用浏览器打开
3. 或部署到任何静态文件服务器

仪表板会自动加载 `data.json`；如果加载失败，会降级为内置模拟数据（FALLBACK）预览。
